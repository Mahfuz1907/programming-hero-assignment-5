Question-1: What are the different ways to select an element in the DOM?

Selecting elements in the DOM (Document Object Model) is fundamental for manipulating web pages. There are several methods to achieve this. getElementById selects an element by its unique ID, making it ideal for targeting specific elements. getElementsByClassName retrieves all elements with a given class name, while getElementsByTagName collects elements with a specified tag name. More versatile methods include querySelector, which selects the first element that matches a CSS selector, and querySelectorAll, which gathers all matching elements.

Question-2: What is the difference between innerHTML, innerText, and textContent?

innerHTML, innerText, and textContent are properties used to retrieve or set the content of HTML elements, but they function differently. innerHTML allows you to access or modify the HTML content of an element, including tags and nested elements, making it powerful but potentially risky if not handled correctly due to XSS (Cross-Site Scripting) vulnerabilities. innerText retrieves or sets only the visible text within an element, considering CSS styling like display: none. Meanwhile, textContent returns or sets the text content of an element and all its descendants, ignoring any CSS styling but preserving the content even if itâ€™s hidden.

Question-3: What is event delegation in the DOM?

Event delegation is a technique where a single event listener is added to a parent element to manage events for multiple child elements. This approach leverages the concept of event propagation (specifically, event bubbling) in the DOM. When an event like a click occurs on a child element, the event bubbles up to the parent element, where it can be captured and processed. This technique is efficient because it minimizes the number of event listeners and allows dynamic handling of elements, even those added to the DOM after the event listener is registered.

Question-4: What is event bubbling in the DOM?

Event bubbling is a process in the DOM where an event starts at the deepest target element and then bubbles up through its ancestor elements. For instance, if you click on a child element within a nested structure, the click event triggers on the child first and then sequentially on its parent, grandparent, and so on, until it reaches the root element. This bubbling mechanism allows higher-level elements to respond to events occurring in their descendants, facilitating techniques like event delegation.

Question-5: How do you create, add, and remove elements using JavaScript?

Creating, adding, and removing elements in the DOM with JavaScript involves several methods. You can create a new element using document.createElement, specifying the type of element you need. Once created, this element can be added to the DOM using methods like appendChild to add it as the last child of a parent element or insertBefore for more specific positioning. Removing elements is straightforward with the remove method for direct removal or removeChild when you need to remove a specific child from its parent element. These methods allow dynamic manipulation of web pages, enabling the creation and deletion of content in response to user actions or other events.
